## 一、需求梳理与功能边界定义

### 1.1 现有素材库模块技术分析

**存储路径**：
- 物理文件存放于 `app.getPath('userData')/materials/` 目录
- 文件通过 UUID 重命名，原文件名保留在 `materials.file_name` 字段
- 渲染进程通过 `material-file://{id}/` 自定义协议（`protocol.handle`）读取图片
- 参见：`src/services/material.service.ts:import()`, `src/main/index.ts:222`

**格式支持**：
- 当前仅支持 `.png` / `.jpg` / `.jpeg` / `.webp` 四种格式
- 参见：`src/ui/MaterialPanel.tsx:67`, `src/services/material.service.ts`

**核心接口**（IPC 桥接层）：

| 通道 | 方向 | 参数 | 返回值 |
|---|---|---|---|
| `material:openDialog` | Renderer → Main | 无 | `string[]` (文件路径) |
| `material:import` | Renderer → Main | `paths: string[]` | `Material[]` |
| `material:list` | Renderer → Main | 无 | `Material[]` |
| `material:delete` | Renderer → Main | `id: string` | `void` |

**数据库表结构**：

```sql
-- materials 表（已有）
CREATE TABLE materials (
  id TEXT PRIMARY KEY,
  file_name TEXT NOT NULL,
  file_path TEXT NOT NULL,   -- 物理文件路径
  mime_type TEXT NOT NULL,
  file_size INTEGER NOT NULL,
  width INTEGER,
  height INTEGER,
  created_at INTEGER NOT NULL
);

-- task_materials 关联表（已有，当前未使用）
CREATE TABLE task_materials (
  task_id TEXT NOT NULL,
  material_id TEXT NOT NULL,
  PRIMARY KEY (task_id, material_id),
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
  FOREIGN KEY (material_id) REFERENCES materials(id) ON DELETE CASCADE
);
```

**关键发现**：
- `task_materials` 关联表已经存在，`MaterialStore.linkTask()` / `unlinkTask()` / `getByTaskId()` 方法已实现但从未被调用——这意味着数据库层天然支持任务与素材的绑定关系，只需在业务层打通即可
- `MaterialPanel` 已实现拖拽导入到素材库（`onDragOver`/`onDrop`），但**不支持从素材库向外部拖出**（`draggable` 未设置）

### 1.2 新增功能核心边界

四个核心模块：

| 模块 | 边界说明 |
|---|---|
| **素材库拖拽至输入框** | 从 MaterialPanel 的缩略图卡片拖出 → 拖入 TaskPanel 的 prompt 输入区域 |
| **参考图临时存储与校验** | 拖入后存入 `task_materials` 关联表；校验格式、尺寸、数量 |
| **参考图与文本任务绑定提交** | 任务创建时关联 `materialIds`，提交至 Runway 时携带图片数据 |
| **适配 Runway 平台参数** | 调研 Runway 多模态 API，在 `runway.adapter.ts` 中实现图片上传/引用 |

## 二、交互流程设计

### 2.1 素材库拖拽增强

**2.1.1 拖出源（MaterialPanel）**

修改 `MaterialPanel.tsx`，为每个缩略图卡片添加 `draggable="true"` 属性：

```tsx
// 缩略图卡片增加：
draggable={true}
onDragStart={(e) => {
  // 设置拖拽数据：传递 material.id 列表（支持多选批量拖拽）
  const ids = selectedIds.has(mat.id) 
    ? Array.from(selectedIds) 
    : [mat.id];
  e.dataTransfer.setData('application/x-runway-material-ids', JSON.stringify(ids));
  e.dataTransfer.effectAllowed = 'copy';
  
  // 悬浮预览：使用缩略图本身作为拖拽预览
  const preview = e.currentTarget.querySelector('img')?.cloneNode(true) as HTMLElement;
  preview.style.width = '80px';
  preview.style.height = '80px';
  preview.style.position = 'absolute';
  preview.style.top = '-1000px';
  document.body.appendChild(preview);
  e.dataTransfer.setDragImage(preview, 40, 40);
  setTimeout(() => preview.remove(), 0);
}}
```

**约束规则**：
- 单次拖拽最多 5 张图片（超限提示 "单次最多添加 5 张参考图"）
- 单张图片上限 20MB（在 `dragStart` 时检查 material.fileSize）
- 拖拽中素材库区域显示半透明蓝色叠加、文字提示 "拖拽到任务面板的输入区域"

**2.1.2 拖入目标（TaskPanel prompt 输入区）**

修改 `TaskPanel.tsx`，为 prompt textarea 包裹一个可接收 drop 的容器：

```tsx
// 输入区域容器：
onDragOver={(e) => {
  e.preventDefault();
  e.dataTransfer.dropEffect = 'copy';
  setDragOverInput(true);  // 高亮边框
}}
onDragLeave={() => setDragOverInput(false)}
onDrop={(e) => {
  e.preventDefault();
  setDragOverInput(false);
  const raw = e.dataTransfer.getData('application/x-runway-material-ids');
  if (!raw) return;
  const ids: string[] = JSON.parse(raw);
  // 校验 + 添加到参考图列表
  addReferenceImages(ids);
}}
```

**视觉反馈状态**：

| 状态 | 视觉表现 |
|---|---|
| 拖拽悬停在输入区上方 | 输入区边框变蓝色虚线，出现 "释放以添加参考图" 提示 |
| 放置成功 | 参考图以缩略图卡片形式出现在输入区下方，短暂绿色闪动 |
| 放置失败（格式/数量超限） | 输入区短暂红色闪烁 + toast 提示具体原因 |

## 三、技术可行性预研

### 3.1 Runway 多模态 API 调研

**调研结论**（基于 Runway Web 端实际交互分析）：

Runway 的视频生成页面支持以下方式上传参考图：
1. **上传首帧图（First Video Frame）**：页面有明显的 "First Video Frame" / "Upload" 区域，接受图片拖拽或点击上传
2. **参考图作为 prompt 补充**：部分模型支持以图生视频

**在 BrowserView 中的实现方案**：

由于 Runway 没有公开 REST API，所有交互通过 BrowserView DOM 操作完成。参考图上传需：

```
方案 A: 模拟用户上传流程（推荐）
  1. 找到 Runway 页面的 "First Video Frame" / "Upload" 区域
  2. 通过 sendInputEvent 点击该区域触发文件选择
  3. 或者通过 JS 构造 File 对象 + DataTransfer 模拟拖拽上传
  
方案 B: 直接操作 DOM 文件输入（备选）
  1. 查找页面中隐藏的 <input type="file"> 元素
  2. 通过 JS 设置其 files 属性
  3. 派发 change 事件触发上传
```

**技术参数（需实机验证）**：

| 参数 | 预期值 | 来源 |
|---|---|---|
| 支持格式 | PNG, JPG, JPEG, WebP | Runway 文档 + 页面 accept 属性 |
| 推荐分辨率 | ≤ 4096×4096 | 保守估计 |
| 文件大小上限 | ≤ 20MB | 保守估计 |
| 单次上传数量 | 1 张（首帧）/ 可能有批量 | 需实际测试 |
| 传输方式 | 页面内直传（非独立 API） | BrowserView DOM 操作 |

### 3.3 图片预处理需求评估

**需要新增的能力**：

| 能力 | 必要性 | 实现方式 |
|---|---|---|
| 格式转换（PNG→JPG） | 中 | Runway 支持 PNG，一般不需转。使用 `sharp` 库（已内置在 Electron 中） |
| 分辨率压缩 | 高 | 超过 4096px 的图需要等比缩放。`sharp.resize()` |
| 文件大小检查 | 高 | 纯逻辑，读取 `material.fileSize` 即可 |
| 水印清除 | 低 | 不在功能范围内 |

**技术风险**：

| 风险 | 等级 | 缓解措施 |
|---|---|---|
| Runway 页面 DOM 结构变更 | 高 | 选择器集中在 `runway.selectors.ts`，单文件修改即可适配 |
| BrowserView sandbox 限制文件访问 | 中 | `material-file://` 协议在主进程注册，BrowserView 可直接加载 |
| 大图拖拽性能 | 低 | 拖拽预览使用 80px 缩略图，不传输原图 |
| 并发任务图片隔离 | 低 | 每任务独立 `materialIds`，不共享 |

### 3.4 跨域与资源权限

- `material-file://` 协议在主进程注册，无跨域问题
- BrowserView 的 `webContents.session` 与主窗口共享 cookie，登录态一致
- 图片通过 `protocol.handle` 返回 `net.fetch(file://...)` 的 Response，无需额外权限
